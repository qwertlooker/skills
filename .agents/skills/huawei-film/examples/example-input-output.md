# 华为胶片使用示例

## 示例输入

> 帮我设计一套华为风格胶片，主题是“AI赋能智能制造”，面向制造企业CIO汇报，5页左右。先输出大纲，再生成HTML。客户尚未提供基线数据和Logo。

## 第一步：任务定义

- **受众：** 制造企业CIO及生产、质量负责人
- **核心利益：** 业务收益、实施风险、数据与系统集成可行性
- **认知水平：** 理解数字化，但不需要算法细节
- **可能态度：** 认可AI方向，担心价值难验证、项目难运营
- **汇报目标：** 使客户同意选择一个可量化场景启动诊断和试点设计
- **约束：** 5页、HTML、无授权Logo、无客户基线数据

## 第二步：故事线

主线：高价值问题 → 闭环方案 → 衡量方法 → 试点路径 → 决策请求。

整套结论：

> 先用一个有基线、有责任人、有验收口径的场景验证闭环，再决定是否规模复制。

## 第三步：逐页结构

### 第1页：封面

- 标题：AI赋能质量与生产优化方案
- 副标题：从单点模型走向可运营业务闭环
- 不放置仿造Logo或密级；保留经授权素材区域

### 第2页：问题与优先级

- 结论标题：质量波动与换型损失是首轮诊断需要验证的高价值问题
- 证据：客户实际基线、损失构成、流程瓶颈
- 未知数据使用占位符：`[客户基线]`、`[统计周期]`、`[数据来源]`

### 第3页：解决方案

- 结论标题：感知—模型—执行—反馈闭环决定AI能否转化为生产改进
- 主体：四层架构或闭环流程
- 3秒摘要：把模型输出嵌入质量处置和工艺调整，而不是停留在独立看板

### 第4页：验证方法

- 结论标题：试点应同时验证模型指标、流程指标和运营责任
- 维度一：模型准确率、误报率、漏报率
- 维度二：处理周期、人工复核量、停线或返工影响
- 维度三：数据、模型、流程分别由谁持续运营
- 目标值保留占位符，禁止编造“99.5%准确率”“30%效率提升”等数据

### 第5页：决策与下一步

- 结论标题：先完成两周诊断并冻结验收口径，再进入试点建设
- 决策请求：确认试点场景、业务负责人、数据负责人和诊断窗口
- “两周”仅在项目计划已有依据时保留，否则改为`[诊断周期待确认]`

## HTML输出要求

- 完整HTML产物参见 `templates/slide-template.html`；本示例重点展示从输入到页面结构的推导，避免与模板重复维护。
- 使用 `resources/huawei-film-theme.css`。
- 使用16:9页面、白色主背景和少量`#C7000B`强调。
- 不显示“HUAWEI”文字Logo，不自动写入`Huawei Confidential`。
- 所有示例数值、组织名、数据源和日期必须替换或明确标为占位符。
- 打印为PDF时移除浏览器预览阴影和页面间距。

## 自检

- 每页标题都是可验证的结论，而不是主题词。
- 每页只有一个主要观点，证据与标题直接相关。
- 未知数据没有被“补全”。
- 技术指标没有无依据换算为ROI。
- 无授权Logo、组织名和密级。
- 复杂图具有一句3秒摘要。
