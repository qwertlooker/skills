# 华为胶片（Huawei Film）设计与制作技能

## 描述

本技能用于生成符合华为官方VI规范的"胶片"（即PPT演示文稿）。华为内部将PPT称为"胶片"，其核心特征是**结构化表达、克制美学、工业级标准**——演示不是信息堆砌，而是价值交付；不是技术炫技，而是以听众为中心的认知协同过程。

输出形式：根据用户需求，可生成以下产物之一：
- **HTML/CSS 网页版胶片**（单页滚动式，可用于浏览器展示）
- **Markdown 结构化大纲**（配合 Marp/Slidev 等工具转换为PPT）
- **PPT 制作规范说明书**（供用户在PowerPoint中手动制作时参考）
- **python-pptx 代码**（直接生成 .pptx 文件）

## 何时使用

当出现以下场景时激活本技能：
1. 用户明确提到"华为胶片"、"华为PPT"、"华为风格演示"、"华为汇报"
2. 用户需要制作企业级商务汇报PPT，且要求专业、结构化、符合华为规范
3. 用户需要将内容按照华为"目标—结构—内容—表达"四维框架进行重组
4. 用户提到需要"浅色主题"或"深色主题"的华为模板风格

## 核心设计规范（强制执行）

### 一、配色方案

**硬性约束：同一页面内颜色不超过4种。**

#### 主色体系（经典华为红·浅色主题）
| 用途 | 色值 (RGB) | HEX | 说明 |
|------|-----------|-----|------|
| 华为红（主色/标题/强调） | R153, G0, B0 | #990000 | 标题、核心强调、品牌标识 |
| 深黑（正文） | R0, G0, B0 | #000000 | 正文文字（避免纯黑，可用R51G51B51即#333333） |
| 深灰（二级正文） | R89, G89, B89 | #595959 | 二级正文、注释说明 |
| 浅灰（背景/分割线） | R242, G242, B242 | #F2F2F2 | 背景色、区块底色、分割线 |
| 白色（页面底色） | R255, G255, B255 | #FFFFFF | 页面主背景 |

#### 辅助配色方案（同页仅选一组）
华为提供13组浅色系配色参考，常用组合：
- **商务蓝组**：主色#003366（深蓝）+ 辅色#4D79A5（中蓝）+ 浅底#E8EEF4
- **科技灰组**：主色#404040（炭灰）+ 辅色#808080（中灰）+ 浅底#F5F5F5
- **活力橙组**：主色#CC6600（深橙）+ 辅色#FF9933（亮橙）+ 浅底#FFF5E6（仅用于数据强调，不可大面积使用）
- **自然绿组**：主色#336633（深绿）+ 辅色#669966（中绿）+ 浅底#EBF0EB
- **优雅紫组**：主色#4D3366（深紫）+ 辅色#7A5C99（中紫）+ 浅底#EDE8F2

#### 深色主题（备选）
- 背景：R0, G0, B0 (#000000) 或 R26, G26, B26 (#1A1A1A)
- 标题文字：反白（白色#FFFFFF），华为红#990000用于强调
- 正文文字：R204, G204, B204 (#CCCCCC)
- Logo红色标志不变

### 二、字体规范

| 元素 | 中文字体 | 英文字体 | 字号 | 字重 |
|------|---------|---------|------|------|
| 中文标题 | 微软雅黑 / 黑体 | - | 30~32pt | 加粗 |
| 英文标题 | - | Arial / FrutigerNext LT Medium | 32~35pt | Medium |
| 一级正文/目录 | 微软雅黑 / 黑体 | Arial | 20~22pt | 常规 |
| 二三级子目录 | 微软雅黑 / 细黑体 | Arial | 18pt | 常规 |
| 注释/说明文字 | 微软雅黑 | Arial | 14~16pt | 常规 |
| 数据大字（核心指标） | 微软雅黑 | Arial | 44~60pt | 加粗 |

**字号梯度硬规则**：标题30-32pt → 正文主干20-22pt → 子目录18pt → 注释14-16pt，严格遵循此层级，不可随意跳级。

**Web端映射**（1pt ≈ 1.333px at 96dpi）：
- 标题：40~44px
- 一级正文：27~29px
- 二三级：24px
- 注释：19~21px

### 三、排版布局

#### 页面结构（每页胶片标准元素）
```
┌──────────────────────────────────────────────┐
│ [华为Logo/客户Logo(右上)]                    │  ← 页眉区
│                                              │
│  ┌──────────────────────────────────────┐    │
│  │ 页面标题（华为红#990000，左对齐）    │    │  ← 标题区
│  └──────────────────────────────────────┘    │
│  ──────────────────────────────────────      │  ← 分割线（细灰线）
│                                              │
│          内容区（留白率30%-40%）              │  ← 内容区
│    每页只承载一个核心观点（金字塔塔尖）       │
│                                              │
│                                              │
│  HUAWEI TECHNOLOGIES CO., LTD.  Huawei Confidential │  ← 页脚（如需要）
└──────────────────────────────────────────────┘
```

#### 排版硬规则
1. **一页一观点**：每页胶片只承载一个核心结论（金字塔原理）
2. **文字精简**：每页文字≤35字，关键词加粗变色而非整句高亮
3. **留白原则**：60%内容 + 30%留白 + 10%强调，留白率不低于30%
4. **对齐方式**：标题左对齐，正文左对齐，禁止居中对齐大段文字
5. **Logo位置**：华为Logo或客户/合作伙伴Logo放置在右上角
6. **信息层级**：标题 → 副标题 → 正文 → 图表，层级清晰不超过3层
7. **禁止**：渐变填充、花哨阴影、艺术字、过多动画特效

### 四、图表与逻辑表达

#### 图表规范
- **一图一结论**：每张图表必须有明确结论，结论写在图表标题或摘要框中
- **数据标注**：柱状图须标注基准线与同比/环比值，折线图标注关键拐点
- **流程图**：必须包含状态反馈箭头，逻辑走向清晰
- **组网图/架构图**：
  - 使用标准化符号表示设备（路由器、交换机、服务器等），禁止使用杂乱实物图
  - 难以用符号表现的，用色块替代并注明名称
  - 保持透视一致性，组网线可带透视感构建立体效果
  - 鲜亮色（符号/连接线）与沉稳色（终端/实物）保持平衡

#### 图标使用
- 使用华为标准图标库中的部件化符号
- 数据库/服务器等采用立体部件化组合表示
- 财务数据、市场份额、组织结构等内容采用反白字加阴影效果突出
- 缩放图片时按Shift键选择角部，避免变形

### 五、内容方法论（华为"胶片"精髓）

#### 四维闭环框架
1. **明确目标（Why）**：演示必须始于清晰、具体、可衡量、有受众指向性的目标
   - 错误示例："介绍平台功能"
   - 正确示例："使分管领导在30分钟内认同我方方案在合规性与效率上的不可替代性"

2. **搭建框架（How）**：黄金圈法则 + SCQA结构
   - **Why（战略价值/痛点共鸣）** → 建立情感连接
   - **How（差异化路径）** → 构建专业可信度
   - **What（可验证成果）** → 落地交付
   - 严格遵循**金字塔原理**：结论先行，以上统下，归类分组，逻辑递进
   - 关键结论前设置"认知钩子"（逻辑断点预埋）对抗注意力衰减

3. **内容填充（What）**
   - **视觉认知负荷管理**：每页≤35字，7±2信息组块
   - **数据业务语言转译**：不写"吞吐量提升2.3倍"，写"支撑全省医保实时结算并发从5万跃升至12万"
   - **复杂架构图配套"3秒可读摘要框"**：用一句话概括核心结论
   - **关键词加粗变色**（华为红），不全句高亮

4. **表达结构（演讲类型适配）**
   - **知识性演讲**：概念分层 → 类比具象 → 案例印证
   - **说服性演讲**：立场声明 → 反方预判 → 证据碾压 → 行动召唤
   - **汇报性演讲**：成果概览 → 关键数据 → 问题分析 → 下一步计划

#### 听众三维预测模型
制作前需明确：
- **兴趣维度**：听众最关心什么？（降本/增效/避责/政绩/ROI）
- **理解能力维度**：听众专业背景如何？需要多少铺垫？
- **态度倾向维度**：中立/支持/质疑/竞品偏好？预判并前置回应

## 指令（生成步骤）

当用户请求制作华为胶片时，按以下步骤执行：

### 第一步：需求澄清
向用户确认以下信息（如用户已提供则跳过）：
1. **演示主题**：胶片的核心主题是什么？
2. **目标受众**：面向谁汇报？（决策者/技术专家/客户/内部同事）
3. **演示类型**：知识性/说服性/汇报性/产品发布？
4. **主题色系**：浅色主题（默认）还是深色主题？偏好哪组辅色？
5. **页数/规模**：大概多少页？是否需要封面、目录、过渡页、结尾页？
6. **输出格式**：HTML网页 / Markdown大纲 / python-pptx代码 / 规范说明？

### 第二步：结构设计
按照华为四维框架设计胶片结构：
1. **封面页**：标题（华为红）、副标题、汇报人/日期、华为Logo位
2. **目录页**：结构化章节列表，使用项目符号
3. **内容页**：每页一观点，严格遵循排版规范
4. **过渡页**（可选）：章节标题页，视觉符号承接
5. **结尾页**：核心结论 + 行动指引 + Thanks

### 第三步：内容生成
根据用户提供的素材，遵循以下原则生成内容：
1. 每页标题必须是一个**结论性语句**，不是中性名词
   - 错误："市场分析" → 正确："市场份额连续三年领先，年增长率达23%"
2. 正文使用**要点式表达**，每点不超过2行
3. 数据必须有对比、有来源、有业务含义
4. 使用华为红(#990000)标注核心数字和关键词

### 第四步：视觉输出
根据选择的输出格式生成对应产物：

**若输出HTML/CSS：**
- 使用16:9宽屏比例（宽1920px × 高1080px）
- 引入微软雅黑字体，降级到system-ui
- 每页作为一个section，支持滚动浏览
- 严格应用上述配色、字号、间距规范
- 添加打印样式，支持导出PDF

**若输出Markdown（Marp/Slidev格式）：**
- 使用Marp分隔符`---`分页
- 在frontmatter中指定theme和尺寸
- 使用标准Markdown语法，不依赖复杂HTML

**若输出python-pptx代码：**
- 使用python-pptx库
- 设置幻灯片尺寸为16:9
- 程序化定义字体、颜色、位置

## 禁用清单（绝对禁止）

1. ❌ 使用超过4种颜色的页面
2. ❌ 使用彩虹色、荧光色、渐变色作为主色
3. ❌ 使用宋体、楷体、艺术字等非规范字体
4. ❌ 一页内堆砌多个无关观点
5. ❌ 使用实物照片代替组网图/架构图中的标准符号
6. ❌ 大段文字（超过35字/页）
7. ❌ 花哨的PPT切换动画和特效
8. ❌ 居中对齐大段正文
9. ❌ 使用非华为红的红色作为主标题色（注意华为红是#990000偏暗红，不是鲜红#FF0000）
10. ❌ 数据没有业务含义（只放数字不说明意义）

## 质量检查清单（生成后自检）

生成胶片后，逐项核查：
- [ ] 配色：每页颜色≤4种，标题色为#990000
- [ ] 字体：中文微软雅黑，英文Arial，字号梯度正确
- [ ] 结构：每页一观点，标题是结论性语句
- [ ] 文字：每页≤35字，关键词加粗变色
- [ ] 留白：留白率≥30%，无拥挤感
- [ ] 逻辑：遵循金字塔原理，结论先行
- [ ] 图表：一图一结论，有标注有来源
- [ ] Logo：右上角位置正确
- [ ] 数据：完成业务语言转译，非纯技术指标
- [ ] 禁用项：无渐变/阴影/艺术字/花哨动画
